# Horror Story: Salary Spreadsheet

Nobody panics when you find a security vulnerability.

They panic when Copilot finds the salary spreadsheet.

## The Setup

Small Copilot pilot. 15 users. Finance and HR.

"We'll clean up permissions before the broader rollout."

Someone in Finance asks Copilot: "What's the salary budget for Q2?"

Copilot helpfully returns a file from a nested SharePoint folder. A file with every employee's compensation. Base, bonus, equity. Names attached.

→ Copilot did exactly what it was designed to do. That was the problem.

## The File

Created in 2021. Inherited permissions from a parent folder. 73 people had access.

Nobody knew it was there. Nobody searched for it. Nobody stumbled on it.

Until someone asked Copilot a question.

→ The file had been exposed for 3 years. Copilot just made it findable.

## The Fallout

IT revoked access within the hour. HR and Legal got involved. The pilot paused.

But the damage was done. Three people saw numbers they weren't supposed to see.

→ You can revoke access. You can't revoke knowledge.

## The Lesson

This wasn't a Copilot failure. Copilot worked perfectly.

This was a permission problem that had existed for years. Copilot just revealed it in seconds.

Security through obscurity died the moment AI could search everything you have access to.

→ Copilot doesn't create exposure. It reveals it.

## The Bottom Line

Every organization has files like this. Buried in nested folders. Inherited permissions nobody audited. "Confidential" in the filename but accessible to half the company.

The question isn't IF Copilot will find them.

It's whether you find them first.
